Arrays and hashes are two elements that are used to store data of any type. Hashes and arrays are incredibly useful in programming and are very commonly utilized. Even though, they seem to serve the same function, there are several differences between arrays and hashes, of which, I will discuss.

An array is an integer-indexed collection of objects of any type. Arrays can hold any data type and can even hold arrays! This is what makes them so flexible, even the data types held in a particular array don't even have to match. You can have floats, ints, strings and arrays all within a single array. Array elements are indexed from 0 upwards. The first element in an array starts at 0, the next element at 1 and so forth.

my_array = [ 1, 2, 3, 4, 5 ]

elements = [_0_|_1_|_2_|_3_|_4_]

This can be a little confusing to people just learning arrays, but they will start to make sense once you start using them. The contents of an array can be manipulated using methods or by directly manipulating the index elements.

Hashes are incredibly useful when you have data that are paired together, that's what separates them from arrays. These paired data are called key and value. Hashes are useful when you need to put pairs of data together, whereas an array is only a collection of objects or data. They can be useful for holding information such as that of a person a person or that of an item. See below for an example of a hash:

my_info = {:first_name=>"Piara", :last_name=>"Sandhu", :age=>"29", :hometown=>"Pinole", :nationality=>"American"}

Hashes rely on an object pointing to another object. Essentially, they're used to assign meaning to data.
