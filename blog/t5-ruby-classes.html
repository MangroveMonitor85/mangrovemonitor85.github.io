Object-Oriented Programming was a breakthrough in computer programming in the 1950s and 1960s at MIT's Artificial Intelligence Group. The majority of programming languages used today are object-oriented such as Java, C++, Python, Ruby, Perl and Objective-C and learning object-oriented programming is essential to staying relevant in the today's tech industry. Languages with classes are much more powerful than those without, because classes are some of the power of today's languages reside and are, basically, here to stay.

Classes are used when you want to create objects. Classes also perform a function, be it as simple is finding an area of a rectangle or as complex as garbage collection. Unlike Ruby, not everything is an object in other languages. Objects must be created using classes and it allows these objects to interact with each other. Objects are often used to represent aspects of the real world. For example, with a class you can create an outline of a person. You can create objects for her name, age, dob, marital status, children, job, current address, employement status and credit card numbers. All of these parameters are useful to know for banking or tax software presumably. This person is in effect an instance of a class.

The best way to visualize a class is to think of it as a blueprint. A blueprint of the house is the class and the actual built house is an object or an instance of that class. An object and an instance are the same thing, in fact they can also be called a class object or a class instance. However, to be more specific and instance describes the relationship between an object and a class. So next time you drive through a suburb you can look at all of the houses and imagine them as individual instances of a class!

So let's open the hood and see what's going on inside a class called Record

class Record

def initialize(band_name, album, number_tracks, genre)

@band_name = band_name

@album = album

@number_tracks = number_tracks

@genre = genre

end

def band

puts "The name of the band is #{@band_name}"

end

def album

puts "the name of the albums is #{@album}"

end

def number_tracks

puts "There are #{@number_tracks} on this record"

end

def genre

puts "#{@band_name} is a #{@genre} band"

end

end

fav_record = Record.new('The Pogues', 'Red Roses for Me', 13, 'punk')

fav_record.band

fav_record.album

fav_record.number_tracks

fav_record.genre

We have just created a class called Record which should take in a bunch of details about a record. The first method that we created is the initialize method that essentially sets the parameters equal to the instance variables. These instance variables, in effect, will be used by the class methods.

Class methods are methods that are only exist inside of the class, but can be called on an instantiated object outside of the class. In this class we have four class methods called band, album, number-tracks and genre. These methods are really simple and their function is to simply display each instance variable to give a description of the record object.
